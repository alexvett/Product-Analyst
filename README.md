# Оптимизация телемаркетинговой кампании банка 

Учебный Data Science проект, направленный на оптимизацию стратегии продаж срочных вкладов через колл-центр банка с помощью анализа данных и приоритизации базы для обзвона.
В проекте используется открытый набор данных телемаркетинговых кампаний португальского банка - [Bank Marketing Dataset](https://www.kaggle.com/datasets/henriqueyamahata/bank-marketing).

## Бизнес-контекст и проблематика
Банк проводит телемаркетинговую кампанию: операторы колл-центра звонят клиентам с предложением открыть срочный вклад. 
Текущая стратегия представляет собой «массовый обзвон» по всей доступной базе. Это приводит к серьезным проблемам:
* Высокие финансовые затраты на оплату труда операторов и телефонию.
* Выгорание базы и раздражение у нецелевых клиентов (снижение лояльности к бренду).
* Глубокая убыточность самой кампании продаж.

## Цели и задачи
**Главная цель:** найти способы приоритизировать базу для обзвона, чтобы максимизировать прибыль кампании и кратно снизить стоимость привлечения одного клиента (CAC).

**Выполненные задачи**
1. Очистка данных и грамотное устранение "утечки данных".
2. Расчет и анализ базовой юнит-экономики кампании.
3. Проведение разведочного анализа данных для сегментации и выявления профиля "идеального клиента".
4. Формирование гипотез по оптимизации (введение лимитов на количество звонков).
5. Проектирование A/B-теста для валидации новой стратегии перед раскаткой на весь колл-центр.

## Ключевые метрики
Для оценки эффективности изменений была выстроена следующая иерархия метрик:

* **Первичные метрики:**
  * **ROI (Return on Investment)** - окупаемость кампании.
  * **CAC (Customer Acquisition Cost)** - стоимость привлечения одного клиента с депозитом.
* **Прокси-метрики:**
  * **Client CR (Conversion Rate)** - конверсия уникального клиента в открытый депозит.
  * **Среднее количество звонков на клиента**.
* **Health-check метрики:**
  * **Уровень негатива** - процент клиентов, бросивших трубку или попросивших «больше никогда не звонить».

## Данные и предобработка
В проекте используется датасет телемаркетинговых кампаний (более 41 000 записей и 21 признак), содержащий:
* Социальный профиль (возраст, профессия, образование, семейное положение).
* Финансовый статус (кредиты, ипотека).
* Историю контактов с клиентом и макроэкономические показатели.

Один из важных этапов подготовки - удаление признака `duration` (длительность звонка). Это значение становится известным только после завершения разговора и имеет прямую корреляцию с целевой переменной (согласившийся клиент говорит дольше, оформляя заявку). Анализ с этим признаком привело бы к некорректным результатам.

## Базовая Unit-экономика (As-Is)
Для оценки текущей ситуации были заданы базовые финансовые параметры (стоимость одного контакта = 100 руб., доход с открытого депозита LTV = 1500 руб.). 

Показатели до оптимизации:
* **Охвачено клиентов:** 41 176
* **Сделано звонков:** 105 735
* **Конверсия в покупку (на клиента):** 11.27%
* **Конверсия в покупку (на звонок):** 4.39%
* **Фактический CAC:** 2 279 руб. *(существенно превышает доход с депозита!)*
* **Финансовый результат:** убыток -3 615 000 руб. (**ROI: -34.19%**)

**Вывод:** Текущая стратегия неэффективна. Кампания сжигает деньги, так как делается слишком много "пустых" звонков.

## Дизайн A/B-теста
На основе инсайтов из EDA была предложена новая стратегия обзвона: интеллектуальная приоритизация базы + жесткий лимит контактов (не более 3 звонков одному клиенту).
Спроектирован A/B-тест длительностью в 1 полную неделю (для нивелирования эффектов дня недели и смен операторов).

**Ожидаемые результаты (To-Be) в тестовой группе**
* Выход **ROI** из убытков в плюс (**+30%**).
* Снижение **CAC** в 2 раза (с 2200 руб. до 1100 руб.).
* Рост **Client CR** с 11.3% до 16.1%.
* Радикальное сокращение средних затрат на телефонию.
